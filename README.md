
# QA Workshop Portfolio 🧪

Este repositorio es el portafolio individual de **Francisco Rafael Rodi Encina** para el workshop **"Ingeniero de Pruebas en el Desarrollo Moderno"**. Aquí se documenta el ciclo de vida completo de QA, desde el análisis estratégico inicial hasta la implementación de pruebas automatizadas.


## Estado Actual

---

## 📑 Gestión Estratégica y Análisis (QA Core)

Esta sección representa la base del portafolio, donde se aplica el pensamiento analítico para garantizar que la calidad esté integrada desde el inicio del proyecto.

### 🔍 Product Overview (Visión del Producto)
* **Análisis del Negocio**: Definición clara de los objetivos de la aplicación bajo prueba (JPetStore), identificando los procesos críticos que generan valor.
* **Definición de Alcance**: Identificación de los módulos y funcionalidades que forman parte del ciclo de pruebas, asegurando que los esfuerzos se alineen con las necesidades del usuario.

### ⚠️ Análisis de Riesgos
* **Identificación de Riesgos**: Evaluación de posibles fallos técnicos y lógicos que podrían impactar la experiencia del usuario o la integridad del negocio.
* **Estrategia de Mitigación**: Priorización de las pruebas basadas en el nivel de riesgo detectado. Esto permite que la automatización se centre en las áreas donde un fallo tendría mayor impacto.

### 📋 Diseño de Pruebas (Estándares ISTQB)
* **Casos de Prueba Manuales**: Documentación detallada de escenarios que incluyen precondiciones, pasos de ejecución y resultados esperados, garantizando una cobertura total de los requerimientos.

## 🤖 Automatización de Pruebas

### 1. Interfaz de Usuario (Playwright)
Validación de flujos End-to-End (E2E) utilizando **JavaScript**.
* **Smoke Suite**: Verificación de la disponibilidad del sitio y carga del catálogo.
* **Búsqueda e Inventario**: Validación funcional del motor de búsqueda (escenarios positivos y negativos).
* **Gestión de Sesión**: Pruebas de seguridad y persistencia en el flujo de Login.
* **Algoritmo Dinámico**: Implementación de bucles para automatizar el agregado masivo de productos por cada categoría del sitio de forma eficiente.

### 2. Capa de Servicios (Karate DSL)
* Automatización de pruebas de API para verificar contratos de servicios, códigos de respuesta y validación de esquemas JSON.
